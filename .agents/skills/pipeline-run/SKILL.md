# SKILL: 電話帳 OCR パイプラインの実行と検証

電話帳画像 → CSV(csv.gz) の 3 段階パイプラインを回すときの手順。詳細仕様は `doc/pipeline.md` を先に読む。

## 手順

1. venv を有効化する（`.venv` または `.venv311`）。
2. 段階ごとに小さなサンプル（数ページ）で試す。いきなり全冊処理しない。
   - `src/split_columns.py` : ページ画像 → カラム画像分割
   - `src/ocr_columns.py` : カラム画像 → OCR（AI プロバイダ利用・課金発生）
   - `src/export_csv.py` : OCR 結果 → csv.gz
3. OCR 段階は API 課金が発生する。サンプル確認なしで大量実行しない。バッチ・キャッシュが使えるなら使う。
4. 出力は `output/` 配下に置く。`data/` の原本画像は絶対に変更・削除しない。
5. 精度検証は、既知の正解がある小範囲（doc/ 記載の評価方法）で行い、結果を報告に含める。

## 禁止事項

- `data/` 配下の電話帳画像原本の変更・削除・git 追加
- `config.json` の API キー値を出力・diff・ログへ含めること
- サンプル確認を経ない全データへの一括 OCR 実行
