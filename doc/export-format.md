# Export Format

## 最終成果物

最終成果物は電話帳ごとに 1 つの `csv.gz` とする。

例:

- `output/csv/1963-02-01-大阪市-50音別電話番号簿.csv.gz`

## CSV ヘッダ

```text
電話番号,名前,都道府県,市区町村,町域,番地
```

## 列定義

- `電話番号`
  - 電話帳に記載された電話番号
  - 括弧や全角数字があれば正規化する
- `名前`
  - 個人名または法人名
- `都道府県`
  - 原則 `大阪府`
- `市区町村`
  - 例: `大阪市東区`
- `町域`
  - 町名、丁目を含む
- `番地`
  - 数字とハイフン主体の地番

## JSON からの変換規則

### 現行実装

入力:

- `output/ocr_columns/<電話帳名>/*.ocr.json`

変換対象:

- 各 `segments[*]`
- `phones`, `names`, `addresses`
- `entryHints.phone`, `entryHints.name`, `entryHints.address`
- `entryHints` が不足する場合は `cleanText` または `rawText` から電話番号を抽出し、空白区切りで名前と住所を推定する

出力規則:

- 電話番号は NFKC 正規化し、`※` と末尾の `代` を落とし、括弧をハイフンへ寄せ、0 始まりでない番号には `06-` を付ける
- 正規化後の電話番号が空の entry は CSV に出力しない
- `phones` や `addresses` に複数候補があれば、AI を使わずに複数 CSV 行へ展開する
- 複数電話番号に対して住所候補が足りない場合は、最後の住所候補を下方向へ引き継ぐ
- `indentLevel` が 1 以上の segment は、直前の浅い階層の名前を親として先頭に連結する
- CSV 出力対象は `entryType=directory_entry` の segment のみとする。`ad_block` と `category_heading` は出力しない
- 電話番号だけの segment が直前エントリに続く場合は、直前の名前・住所を引き継いで番号違いの別行として出力する
- 住所は昭和期の大阪市区・周辺自治体の略記辞書で展開する
- `addresses` に複数候補があっても、長い候補が他候補をすべて含む場合は、複数住所ではなく複数行にまたがる 1 つの住所として長い候補 1 件に畳む
- 電話番号・名前がなく住所だけの segment が直前エントリに続く場合は、別行にせず直前住所の折り返しとして連結する
- `possible_ad_entry` はレビュー用フラグであり、CSV 出力可否には使わない
- 本体名、校舎名、複数電話番号、複数住所が 1 segment に入る場合は、`cleanText` 上の出現順から直前の校舎名と直後の住所を電話番号へ対応させる
- 例: `東,内淡路1-28` は `大阪府,大阪市東区,内淡路,1-28`
- 例: `布,中小阪483` は `大阪府,布施市,中小阪,483`
- 例: `三島,三島正音寺` は `大阪府,三島郡三島町,正音寺,`
- 例: `北河,門真二番` は `大阪府,北河内郡門真町,二番,`
- 例: `尼,森160` は `兵庫県,尼崎市,森,160`
- OCR が区切りカンマを落とした場合も、略記辞書の前方一致で展開する。例: `尼若王寺中ノ坪` は `兵庫県,尼崎市,若王寺中ノ坪,`
- 昭和38年2月1日大阪市50音別電話番号簿では、1963-08-01 市制施行前のため `門真` を `北河内郡門真町` として扱う
- 末尾の半角数字とハイフン列を `番地` として分離する
- 6 項目に欠損があれば空文字にする
- 列順は必ず固定する
- 文字コードは UTF-8
- gzip 圧縮して保存する

### 将来の正規化ステージ導入後

入力:

- `output/normalized_entries/<電話帳名>/entries.normalized.json`

変換対象:

- 各 `entries[*].normalized`

出力規則:

- `normalized` に必要 6 項目があればそのまま出力する
- 欠損値は空文字にする
- `reviewFlags` が残るエントリも出力はするが、必要なら別途レビューリストを作る
- 列順は必ず固定する
- 文字コードは UTF-8
- 改行は LF または CRLF のどちらでもよいが統一する
- 最後に gzip 圧縮する

## 非保持情報

CSV では次の情報を持たない。

- 元画像位置
- 断片の連結履歴
- モデル補完の理由
- 信頼度
- レビュー対象フラグ

これらは JSON 正本にのみ保持する。

## 実装上の注意

- CSV を正本にしない
- 同じ電話帳から何度再生成しても列順と正規化規則が変わらないようにする
- 将来 TSV を出す場合も、同じ JSON 正本から派生させる
